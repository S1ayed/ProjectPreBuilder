# 文本框功能设计

日期：2026-03-27

## 1. 设计目标
- 在“批注 > 文字”模式下提供可拖拽到画布的文本框创建能力。
- 文本框支持“画布内编辑 + 侧栏编辑”双入口，并保持双向实时同步。
- 文本框缩放时文字字号按比例缩放，保证视觉一致性。
- 默认状态弱干扰（边框透明），选中状态高可见（深蓝边框 + 明显右下角手柄）。

## 2. 功能范围
- 文字工具下显示“文本框拖拽按钮”。
- 拖拽按钮到画布后创建文本框节点。
- 默认文案为：点击编辑文字。
- 单击文本进入编辑；也支持在左侧文本编辑区修改。
- 删除全部内容后回退显示默认文案。
- 选中文本框显示深蓝边界和右下角缩放手柄。

## 3. 交互设计

### 3.1 侧栏交互（SymbolsSection）
- 当 activeTool = text 时显示：
  - 拖拽按钮：拖拽到画布创建文本框。
  - 文本编辑区：编辑当前选中文本框内容。
- 若当前未选中文本框：
  - 文本编辑区置灰并显示提示。

### 3.2 画布交互（DraggableGridCanvas）
- 文本框初始状态：
  - 边框透明。
  - 文本显示“点击编辑文字”（占位文案）。
- 单击文本框：
  - 进入编辑态。
  - 若当前内容是默认文案，编辑框自动清空。
- 编辑完成后：
  - 内容为空时，展示层显示默认文案。
  - 内容非空时，展示用户输入文本。

### 3.3 选中与缩放
- 选中文本框时：
  - 边框变为深蓝色。
  - 右下角显示明显小型缩放手柄。
- 拖拽缩放手柄：
  - 调整文本框宽高。
  - 字号按缩放比例同步变化（设置最小/最大字号限制）。

## 4. 状态模型

### 4.1 节点数据结构
```ts
shape = {
  id: string,
  type: 'text',
  x: number,
  y: number,
  width: number,
  height: number,
  payload: {
    text: string,      // 为空时展示默认文案
    fontSize: number   // 基础字号，缩放时联动更新
  },
  fillColor: string,
  strokeColor: string,
  strokeWidth: number,
  opacity: number,
}
```

### 4.2 编辑态状态
```ts
editingTextId: string | null
editingTextValue: string
selectedShapeIds: string[]
selectedTextShape: shape | null
```

## 5. 同步策略
- 画布编辑 -> 侧栏：
  - 文本输入变化后立即写回 shape.payload.text。
  - selectedTextShape 由 selectedShapeIds 派生，侧栏同步渲染。
- 侧栏编辑 -> 画布：
  - 左侧 textarea 的 onChange 直接更新当前选中文本节点 payload.text。
- 占位文案策略：
  - 仅在“展示层”处理，避免将占位文案作为真实业务文本长期存储。

## 6. 样式规范
- 非选中：
  - 文本框边框透明。
  - 背景透明，底层网格可见。
- 选中：
  - 边框：深蓝（推荐 #164488）。
  - 阴影：轻微外发光增强可见性。
- 缩放手柄：
  - 16x16 左右，亮蓝填充 + 白边 + 角标。
  - 光标：nwse-resize。

## 7. 边界与容错
- 允许文本为空，展示默认文案。
- 限制最小字号（如 10）与最大字号（如 180）。
- 仅在 type = text 时启用字号联动。
- 删除节点时自动清理选中状态与编辑态。

## 8. 验收清单
- 文字工具下可见文本框拖拽按钮。
- 拖拽到画布可创建文本框。
- 默认显示“点击编辑文字”。
- 单击文本可编辑，默认文案自动清空。
- 清空文本后再次显示默认文案。
- 选中后边框变深蓝，手柄明显可见。
- 拖拽手柄缩放时，文字字号同步缩放。
- 左侧编辑与画布编辑实时同步。
