# 解释器校验规则

## 1. 文档目的

本文档定义 PreBuilder 解释器在导出 JSON 之前必须执行的校验规则，目标是保证：

- 数据结构完整且可解析。
- 依赖关系合法且可追踪。
- 语义字段满足最小生成条件。
- 导出结果可直接作为 LLM 构建项目的输入契约。

## 2. 适用范围

适用于从画布模型（nodes + dependencies）导出最终 JSON 的所有流程。

## 3. 校验分级

- ERROR：阻断导出，必须修复。
- WARN：允许导出，但需要提示用户风险。
- INFO：仅提示优化建议。

## 4. 输入模型假设

解释器输入包含：

```json
{
  "nodes": [],
  "dependencies": [],
  "meta": {}
}
```

其中：

- `nodes[i]` 至少包含：`id`、`kind`、`payload`。
- `dependencies[i]` 至少包含：`id`、`from`、`to`、`type`。
- `kind` 由注册表定义，当前支持：`project`、`directory`、`file`、`config`。

## 5. 校验执行顺序

1. 结构校验（是否有必要字段）。
2. 基础 schema 校验（字段类型、必填项、枚举值）。
3. 图校验（节点/边关系是否合法）。
4. 语义校验（各 kind 的业务约束）。
5. 导出前总结（错误、警告、建议）。

## 6. 规则清单

### 6.1 通用结构规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-001 | ERROR | `nodes` 必须是数组。 |
| PBV-002 | ERROR | `dependencies` 必须是数组。 |
| PBV-003 | ERROR | 所有节点 `id` 必须唯一。 |
| PBV-004 | ERROR | 所有连线 `id` 必须唯一。 |
| PBV-005 | ERROR | 节点 `kind` 必须存在且在 kind 注册表中。 |
| PBV-006 | ERROR | 连线必须包含 `from`、`to`、`type`。 |

### 6.2 拓扑与依赖规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-101 | ERROR | 每条连线的 `from` 节点必须存在。 |
| PBV-102 | ERROR | 每条连线的 `to` 节点必须存在。 |
| PBV-103 | ERROR | 禁止自连接（`from === to`）。 |
| PBV-104 | ERROR | 相同方向的重复连线（同 `from`、`to`、`type`）禁止重复。 |
| PBV-105 | WARN | `import` 类型连线若形成环，给出循环依赖警告。 |
| PBV-106 | WARN | 孤立节点（无入边且无出边）提示检查是否遗漏关系。 |

### 6.3 project 节点规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-201 | ERROR | 文档中必须且只能有 1 个 `project` 节点。 |
| PBV-202 | ERROR | `ProjectName` 必填且为非空字符串。 |
| PBV-203 | ERROR | `ProjectDescription` 必须为字符串（可空）。 |
| PBV-204 | ERROR | `stack` 必须为字符串数组。 |
| PBV-205 | WARN | `stack` 为空时提示技术栈缺失。 |

### 6.4 directory 节点规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-301 | ERROR | `DirectoryName` 必填且为非空字符串。 |
| PBV-302 | ERROR | `DirectoryDescription` 必须为字符串（可空）。 |
| PBV-303 | ERROR | `DirectoryName` 不允许包含非法路径字符：`<>:"|?*`。 |
| PBV-304 | WARN | 未检测到 `src` 目录节点时提示初始化结构缺失。 |

### 6.5 file 节点规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-401 | ERROR | `FileName` 必填且为非空字符串。 |
| PBV-402 | ERROR | `FileType` 必填且为非空字符串。 |
| PBV-403 | ERROR | `Prompt` 必填且必须为对象。 |
| PBV-404 | WARN | `FileName` 不含扩展名时提示检查。 |
| PBV-405 | WARN | `Prompt.generation.goal` 缺失时提示生成目标不明确。 |
| PBV-406 | WARN | `FileType` 未在 FileType 注册表命中时提示。 |

### 6.6 config 节点规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-501 | ERROR | `ConfigName` 必填且为非空字符串。 |
| PBV-502 | ERROR | `ConfigFormat` 必须在 `json/yaml/toml/text` 范围内。 |
| PBV-503 | ERROR | `Directives` 存在时必须为对象。 |
| PBV-504 | ERROR | `Constraints` 存在时必须为字符串数组。 |
| PBV-505 | WARN | `GenerationStrategy=template-driven` 且 `TemplateID` 为空时提示模板缺失。 |

### 6.7 连线语义规则

| Rule ID | 级别 | 规则说明 |
| --- | --- | --- |
| PBV-601 | ERROR | 连线 `type` 不能为空。 |
| PBV-602 | WARN | 未知连线类型（不在白名单）提示兼容风险。 |
| PBV-603 | INFO | 建议优先使用标准类型：`import`、`compose`、`generate-from`。 |

## 7. 校验结果输出格式（建议）

```json
{
  "ok": false,
  "summary": {
    "errorCount": 2,
    "warnCount": 3,
    "infoCount": 1
  },
  "issues": [
    {
      "ruleId": "PBV-201",
      "level": "ERROR",
      "message": "文档中必须且只能有 1 个 project 节点",
      "target": { "kind": "node", "id": "shape-3" }
    },
    {
      "ruleId": "PBV-405",
      "level": "WARN",
      "message": "File 节点缺少 Prompt.generation.goal",
      "target": { "kind": "node", "id": "shape-8" }
    }
  ]
}
```

## 8. 导出门禁策略

- 当 `ERROR > 0`：禁止导出。
- 当 `ERROR = 0` 且 `WARN > 0`：允许导出，但需二次确认。
- 当仅有 `INFO`：直接导出。

## 9. 最低可用校验集合（MVP）

首期必须实现以下规则：

- PBV-001, PBV-002, PBV-003, PBV-004, PBV-005
- PBV-101, PBV-102, PBV-103, PBV-104
- PBV-201, PBV-202, PBV-204
- PBV-301, PBV-401, PBV-402, PBV-403
- PBV-501, PBV-502

以上规则通过后即可保证导出 JSON 在结构上可被稳定消费。
