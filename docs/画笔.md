# 画笔功能设计

日期：2026-03-26

## 1. 设计目标
- 提供高效、直观的批注绘制体验。
- 支持“快速选择 + 精细调节”的颜色与笔触配置。
- 让橡皮擦具备可预览、可控、局部擦除能力。
- 保持与画布现有交互（缩放、平移、选择）的一致性。

## 2. 功能范围
- 画笔模式：自由绘制笔画。
- 橡皮擦模式：按预设形状和尺寸进行局部擦除。
- 颜色配置：快捷色 + 滑动连续变色。
- 粗细配置：多档圆点预设。
- 橡皮擦预设：3 列 x 4 行（形状与大小组合）可视化选择。
- 画布预览：橡皮擦光标实时显示当前尺寸和形状。

## 3. 交互设计

### 3.1 批注面板结构
三列布局：
- 第一列（颜色）：
  - 快捷色块。
  - 填充图形节点预览（展示当前颜色）。
  - 滑动控件（连续变色）。
- 第二列（粗细）：
  - 不同直径圆点按钮（表示笔触宽度）。
- 第三列（橡皮擦）：
  - 3 x 4 独立预设网格。
  - 每个格子是固定形状 + 固定尺寸的图形按钮。

### 3.2 画布行为
- 画笔模式：按当前颜色与粗细实时绘制。
- 橡皮擦模式：
  - 鼠标移动时显示橡皮擦预览轮廓。
  - 按下并移动时只擦除轮廓覆盖区域。
  - 不删除整条笔画，仅移除被命中的笔画段。

## 4. 状态模型
核心状态建议如下：

```ts
penSettings = {
  color: string,         // 当前画笔颜色
  width: number,         // 当前画笔粗细
  mode: 'draw' | 'erase',
  eraserSize: number,    // 当前橡皮擦尺寸
  eraserShape: 'circle' | 'square' | 'diamond'
}
```

笔画数据建议如下：

```ts
stroke = {
  id: string,
  points: Array<{ x: number; y: number }>,
  strokeColor: string,
  strokeWidth: number
}
```

## 5. 擦除算法设计（局部擦除）
- 输入：当前橡皮擦中心点、形状、尺寸、全部笔画。
- 处理：
  - 遍历笔画点序列。
  - 判断点是否落入橡皮擦轮廓。
  - 命中点从序列中剔除，未命中点按连续性切分为多个子段。
- 输出：
  - 未命中的笔画段重新组成新笔画。
  - 命中区域被移除，实现“局部擦除”。

## 6. 可用性与可访问性
- 所有图形按钮保留 `aria-label`。
- 当前选中项需有明显视觉高亮。
- 预设网格在窄屏下可纵向重排，保证触控可点。

## 7. 性能要求
- 画笔与擦除在连续拖动场景下应保持流畅。
- 擦除计算应尽量避免不必要的全量重建。
- 对大画布数据可考虑分段采样或节流策略。

## 8. 后续扩展
- HSV/HSL 双滑块（色相+明度/饱和度）。
- 最近使用颜色与自定义色板。
- 软边橡皮擦（羽化边缘）。
- 吸管取色与撤销/重做联动。
